论文笔记-RDE-VOS论文细节记录

1.用于decoder部分的特征

memory bank中的key

大小：(Ck,H/16,W/16)，H,W为图片原始的高度和宽度

来源：由之前帧的图像经过ResNet50网络结构(使用的是res4的特征)，然后经过一个卷积网络映射得到

memory bank中的value

大小：(Cv,H/16,W/16)，H,W为图片原始的高度和宽度

来源：由将之前帧的图像mask和image进行拼接，然后经过ResNet18网络结构获得(使用的是res4的特征)，然后与memory bank中的key融合，融合办法如下：

  • 拼接key和value特征
  • 将拼接得到的特征输入到两个ResBlocks网络[1]，一个CBAM块[2]
  • 最终得到真正的value

来自查询帧的特征

（1）查询帧的key特征

大小：(C,H/16,W/16)，H,W为图片原始的高度和宽度

来源：查询帧的图像经过ResNet50网络结构(使用的是res4的特征)，然后经过一个卷积网络映射得到

（2）查询帧的value特征

大小：(C,H/16,W/16)，H,W为图片原始的高度和宽度

来源：查询帧的图像经过ResNet50网络结构(使用的是res4的特征)，然后经过一个卷积网络压缩通道维度（由1024变为512），由此得到最终结果

（3）查询帧经过ResNet网络得到的特征（skip connection）

有两个，来自经过ResNet50网络得到的stride=8和stride=4的特征

注意：它的计算其实就是通过key encoder获得，首先获得特征，然后将这个特征再次进行映射就得到了key

方法：L2 Similarity Computation

3.readout过程

这是一个从memory bank中读出有用信息的算法。

输入：来自memory bank中的key(维度：Ck * THW)和value，来自查询帧的key(维度：Cv * HW)和value

过程：

  • 计算memory bank的key和查询帧的key的相似度(affinity)
  • 相似度和memory bank的key相乘，得到的输出维度变为B * Cv * H * W
  • 将上一步得到的输出与查询帧的value进行拼接（在通道维度上），这就是最后的readout特征

4.多个物体进行分割

value encoder:value encoder本质上没有变化，只是将之前帧的图像的所有物体的mask和image进行拼接,注意主体的mask在前面

5.训练过程

The model is trained progressively with different stages (0: static images; 1: BL30K; 2: 300K main training; 3: 150K main training). After each stage finishes, we start the next stage by loading the latest trained weight.
